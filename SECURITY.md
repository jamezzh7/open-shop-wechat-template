# 安全政策

Open Shop 是微信小程序电商模板，代码会被不同商家部署到各自的微信小程序、CloudBase 环境和支付配置中。请不要在公开 Issue、PR、评论或截图中暴露 appid、CloudBase 环境 ID、访问密钥、支付商户信息、Webhook、数据库连接信息或真实用户数据。

## 支持范围

当前安全维护范围为 `main` 分支的最新代码。

历史分支、私有二次开发版本、商家本地配置文件和客户自行修改的部署环境不在本仓库统一安全维护范围内。使用方应及时同步最新模板代码，并在自己的 CloudBase、微信支付和后台账号体系中执行安全配置。

## 报告漏洞

如果你发现安全问题，请优先通过 GitHub 的私密漏洞报告入口提交：

<https://github.com/jamezzh7/open-shop-wechat-template/security/advisories/new>

报告中建议包含：

- 受影响的目录、文件或接口。
- 复现步骤和影响说明。
- 你已经验证过的环境信息。
- 可行的修复建议或临时缓解方式。

请不要提交包含真实密钥、真实订单、真实用户资料或支付凭证的附件。需要演示时，请使用脱敏数据或本地测试环境数据。

如果私密漏洞报告入口不可用，请先创建一个不包含漏洞细节和敏感信息的公开 Issue，说明需要安全联系渠道。

## 处理预期

- 维护者会尽快确认报告是否可复现。
- 对高风险问题，会优先给出修复或临时缓解建议。
- 在修复发布前，请不要公开传播可被直接利用的细节。

## 安全配置提醒

使用本模板上线前，请至少确认：

- `project.config.json`、`cloudbaserc.json`、`admin-web/.env.local` 和 `shops/*.local.json` 只包含自己的环境信息，并且不会提交到 Git。
- CloudBase 数据库、云函数、静态托管和 Web 用户权限符合最小权限原则。
- 微信支付商户号、证书、回调地址和支付云模板只在自己的控制台中配置。
- 企业微信 webhook、CloudBase token、数据库连接信息和其他凭证通过环境变量或控制台配置，不写入源码。
- 后台管理员绑定只授予可信账号，离职或交接时及时撤销权限。
- 发布前执行 `git status --short` 和必要的密钥扫描，确认没有误提交本地配置或敏感信息。
